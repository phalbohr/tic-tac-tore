# Educational Log: Phase 1 - Backend Statistics Calculation & APIs
## Task: Implement Statistics DTOs and Repository Queries

### Введение
В рамках этой задачи мы закладываем фундамент для системы статистики и лидербордов. Нам необходимо определить структуру данных, которую мы будем возвращать пользователю (DTO), и реализовать эффективные запросы к базе данных для агрегации данных из истории матчей.

### Резюме шага: Инициализация задачи
Мы начали с обновления плана реализации (`plan.md`) и создания этого журнала. Это важный этап профессиональной разработки — фиксация намерений и подготовка к документированию процесса.

### Резюме шага: Проектирование DTO и базовая логика (Green Phase)
Мы реализовали структуру DTO с использованием Java Records. Records идеально подходят для передачи данных (DTO), так как они неизменяемы и лаконичны.

#### Ключевые решения:
1.  **Java Records**: Вместо классических классов с Lombok `@Data` мы использовали `record`. Это современный стандарт в Java для простых контейнеров данных.
2.  **Поле `winner` в `Match`**: Мы решили добавить поле `winner_team` в сущность `Match`. Хотя теоретически победителя можно вычислить динамически из списка игр, хранение результата в самой сущности критически важно для производительности SQL-запросов (агрегации, лидерборды).
3.  **TDD (Test-Driven Development)**: Мы сначала написали тест `StatisticsServiceTest`, который не компилировался (Red), а затем реализовали логику в `StatisticsService` (Green). Это гарантирует, что наш код делает именно то, что от него ожидается.

#### Педагогическая заметка:
Обратите внимание на метод `calculateWinRate`. При работе с числами с плавающей точкой важно правильно обрабатывать деление на ноль и округление. Мы используем `Math.round(value * 10000.0) / 100.0` для получения процента с двумя знаками после запятой (например, `66.67`).
