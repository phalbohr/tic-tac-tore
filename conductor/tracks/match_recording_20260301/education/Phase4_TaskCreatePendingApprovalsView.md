# Образовательный журнал: Фаза 4, Задача: Create Pending Approvals View

## Введение
В рамках этой задачи мы реализуем интерфейс для просмотра матчей, ожидающих подтверждения от текущего пользователя. Это критически важная часть системы подтверждения (Confirmation Workflow), обеспечивающая честность и точность данных.

## Архитектурные решения и ментальные модели
### 1. Состояние матча и фильтрация
На стороне фронтенда нам необходимо отображать только те матчи, где:
- Статус матча `PENDING_APPROVAL`.
- Текущий пользователь является одним из оппонентов (не создателем и не напарником создателя).

## Knowledge Transfer

### Оптимизация запросов (N+1 Problem)
При работе с JPA и Hibernate часто возникает проблема N+1: когда для списка объектов (например, матчей) выполняются дополнительные SQL-запросы для каждой связанной сущности (создателя, участников, игр).
**Решение:** Мы использовали `JOIN FETCH` в JPQL-запросе в `MatchRepository`. Это заставляет Hibernate загрузить все необходимые данные одним сложным SQL-запросом с JOIN-ами, что на порядок быстрее при больших объемах данных.

### Принцип "Tell, Don't Ask"
Вместо того чтобы сервис проверял статус матча и менял его вручную (`match.setStatus(...)`), мы вынесли эту логику в саму сущность `Match` (методы `approve()` и `reject()`).
**Почему это важно:** Это инкапсулирует бизнес-правила внутри доменной модели. Теперь никто не сможет перевести матч в статус `CONFIRMED`, не проверив, был ли он в статусе `PENDING_APPROVAL`, так как логика проверки находится внутри метода `approve()`.

### Разделение ответственности (Interface-Driven Documentation)
Для чистоты кода в контроллере мы вынесли все аннотации OpenAPI/Swagger в отдельный интерфейс `MatchApi`.
**Результат:** `MatchController` теперь содержит только чистую реализацию логики, а документация API хранится централизованно в интерфейсе, что упрощает чтение и поддержку кода.

### Мобильный UX
Интерфейс `PendingApprovals.vue` спроектирован с использованием принципов "Mobile First": крупные кнопки действий (44px+), четкое разделение команд цветом и карточная структура, которая хорошо адаптируется под узкие экраны смартфонов.
