# Обучающий журнал: Фаза 3, Задача: Добавление ограничений для полей ввода счета

## Описание задачи
Добавление атрибута `max="99"` для полей ввода счета в компоненте `MatchScoring.vue` для предотвращения ввода некорректных данных и улучшения пользовательского интерфейса.

## Перенос знаний

### 1. Валидация на стороне клиента (HTML5 Attributes)
Использование атрибутов `min` и `max` в полях ввода типа `number` — это первый эшелон защиты и улучшения UX. 
- **UX (User Experience):** Браузеры могут отображать специальные элементы управления (стрелочки вверх/вниз), которые будут учитывать эти границы.
- **Предотвращение ошибок:** Пользователь сразу видит, что ввод трехзначного числа (например, 100) не предусмотрен логикой игры (в настольном теннисе или подобных играх счет редко превышает 99).

### 2. Тестирование атрибутов компонентов
В профессиональной разработке мы тестируем не только логику (функции), но и важные атрибуты DOM-элементов, которые влияют на поведение системы.
В файле `MatchScoring.spec.ts` мы добавили тест:
```typescript
it('has max score limit of 99 on input fields', () => {
  const wrapper = mount(MatchScoring, {
    props: { players: mockPlayers }
  })

  const scoreInputs = wrapper.findAll('input[type="number"]')
  scoreInputs.forEach(input => {
    expect(input.attributes('max')).toBe('99')
  })
})
```
Этот тест гарантирует, что если кто-то случайно удалит атрибут `max` при рефакторинге, автоматизированная сборка (CI) упадет, и ошибка будет замечена.

### 3. Чистота кода и стандарты
Согласно нашему Workflow, мы удаляем все "учительские" комментарии из кода перед коммитом. Код должен быть самодокументированным. Имена атрибутов (`min`, `max`) и констант должны быть понятны без дополнительных пояснений.

## Ментальная модель инженера
Инженер не просто "добавляет поле", он думает о **границах данных** (boundary values). Если есть счет, может ли он быть отрицательным? (Нет, `min="0"`). Может ли он быть бесконечным? (Нет, разумный предел — `max="99"`). Установка таких границ на раннем этапе (в UI) экономит ресурсы сервера и упрощает обработку ошибок.
