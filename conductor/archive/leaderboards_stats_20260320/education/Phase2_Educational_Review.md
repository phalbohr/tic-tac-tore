# Обучающий журнал: Создание страницы лидеров и сервиса статистики

## Введение
В рамках этой задачи мы реализуем клиентскую часть для отображения глобальных рейтингов игроков. Мы создадим сервис для взаимодействия с API статистики и Vue-компонент для отображения таблиц лидеров с поддержкой пагинации.

## Архитектурные решения и ментальные модели
При проектировании фронтенд-компонентов для работы с табличными данными важно учитывать:
1. **Разделение ответственности (Separation of Concerns):** Логика запросов к API должна быть инкапсулирована в сервисе (или store), а компонент должен отвечать только за отображение и обработку действий пользователя.
2. **Типизация:** Использование TypeScript для описания DTO (Data Transfer Objects) позволяет избежать ошибок при работе с данными, приходящими с сервера.
3. **Реактивность:** Мы будем использовать Composition API для управления состоянием пагинации и текущего выбранного типа рейтинга.

В ходе выполнения задачи мы реализовали:
1. **`statisticsService.ts`**: Сервис для работы с API статистики. Мы использовали `URLSearchParams` для чистого формирования Query-параметров. Также мы интегрировали `AbortSignal` для поддержки отмены запросов, что критично для предотвращения Race Conditions во фронтенде.
2. **`LeaderboardView.vue`**: Компонент отображения рейтингов.
    - Использовали вкладки (Tabs) для переключения между типами рейтинга (Overall, Attacker, Defender).
    - Реализовали пагинацию (кнопки Next/Previous и выбор размера страницы) с автоматическим обновлением данных через `watch`.
    - Добавили обработку состояний загрузки (Loading) и ошибок (Error) с кнопкой Retry.
    - Интегрировали `AbortController` для автоматической отмены незавершенных запросов при смене вкладок или страниц.

**Совет для студента:** Всегда думайте о том, что произойдет, если пользователь будет быстро переключать фильтры или страницы. Без `AbortController` старые запросы могут выполниться позже новых и заменить актуальные данные неверными.
# Обучающий журнал: Реализация фильтров периода и порога

## Введение
В этой задаче мы расширяем функционал страницы лидеров, добавляя возможность фильтрации данных по времени (неделя, месяц, год, всё время) и установки минимального порога матчей для попадания в рейтинг.

## Архитектурные решения
1. **Расширение состояния:** Мы добавим новые реактивные переменные для периода и порога в `LeaderboardView.vue`.
2. **Связанные фильтры:** Важно обеспечить сброс пагинации при изменении любого фильтра, чтобы пользователь всегда начинал просмотр с первой страницы отфильтрованных результатов.
3. **UX-фильтрация:** Минимальный порог матчей позволяет отсеивать случайных игроков из глобального рейтинга, делая его более соревновательным.
